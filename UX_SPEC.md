# UX Specification

## Product stance

WSIS should feel like an honest narrowing tool.

The UX should repeatedly reinforce three ideas:

- Explore first
- Inspect before comparing
- Trust what is source-backed

## Home / Discovery Map

- Show a map-first exploration experience
- Rank and shade cities using only the trust-first MVP score
- Keep weight controls for affordability, job market, safety, and climate
- Present social sentiment as context-only, not as a ranking control
- Explain in plain language what the score includes
- Show trust-first messaging in hero copy and side panel copy

## City Profile

- Show the city score, breakdown, and plain-language inclusion/exclusion rules
- Show confidence and provenance for every dimension
- Show beta warnings if a city is inspection-only
- Keep the social panel visible with an explicit “not included in score” label

## Comparison

- Compare cities side by side using the same trust-first score semantics as Home
- Show a trust summary row for each city
- Keep social sentiment visible as context only

## Design rules

- Never imply seeded Reddit context is rank-defining
- Never imply an estimated city is part of ranked discovery if it is not MVP-eligible
- Prefer readable explanations over dense control surfaces
