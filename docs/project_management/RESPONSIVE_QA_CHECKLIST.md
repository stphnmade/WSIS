# WSIS Responsive QA Checklist

## Viewports

- iPhone 12 / 390 x 844
- Small mobile / 360 x 740
- Tablet / 768 x 1024
- Desktop / 1440 x 900

## Checks

| Area | Requirement | Status |
| --- | --- | --- |
| Forms | Single-column on mobile with visible labels. | Pending |
| Primary action | One obvious primary action per screen. | Pending |
| Progress indicators | Show only step start/end/state, no duplicate explanatory text. | Pending |
| Tables | Convert dense tables to cards or concise summaries on mobile. | Pending |
| Map | Markers and labels do not overlap badly on mobile. | Pending |
| Text | No clipped labels, overflowing buttons, or duplicated copy. | Pending |
| Touch targets | Controls are at least 44px where practical. | Pending |
| Evidence panels | Secondary detail is hidden until requested. | Pending |

## Proof Case

David must be able to get a verdict without reading raw methodology.

Sarah must be able to challenge the verdict and find weak evidence without searching through duplicated UI text.
